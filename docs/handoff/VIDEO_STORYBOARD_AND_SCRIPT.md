# PURE-JADE 演示视频脚本与分镜

建议视频时长：5-8 分钟。

目标：让老师即使不运行代码，也能看懂项目价值、系统功能、模型使用方式和测试效果。

## 视频结构总览

| 段落 | 建议时长 | 内容 |
|---|---:|---|
| 1. 项目背景 | 45-60 秒 | 为什么做情绪支持系统 |
| 2. 系统架构 | 60-90 秒 | 四阶段链路与 Direct baseline |
| 3. 前端演示 | 2-3 分钟 | 输入、运行、查看卡片 |
| 4. A/B 对比 | 1-2 分钟 | Direct vs PURE-JADE v0.26 |
| 5. 总结与分工 | 45-60 秒 | 亮点、局限、成员贡献 |

## 分镜 1：项目背景

画面建议：

- 项目标题页；
- 或打开论文首页/README；
- 或展示前端窗口标题。

讲稿：

```text
大家好，我们的小组项目是 PURE-JADE，一个面向大学生情绪支持场景的模块化大语言模型原型系统。我们关注的是学习压力、小组合作、人际比较、自我怀疑等低风险情绪支持场景。相比普通聊天机器人，我们希望系统不仅能生成回复，还能记录模型是如何理解用户状态、选择共情策略并生成最终回应的。
```

## 分镜 2：系统架构

画面建议：

- 展示流程图；
- 可以用论文里的架构图；
- 或展示 `README_FOR_GROUPMATES.md` 里的流程。

讲稿：

```text
系统整体采用多阶段链路。用户输入后，第一部分生成用户状态卡，识别情绪、需求、风险和上下文变化。第二部分生成共情策略卡，决定本轮应该安抚、肯定、澄清还是提供下一步建议。第三部分根据策略卡生成最终给用户看的行为回应。第四部分则用于评价和对比。

为了判断模块化链路是否真的有价值，我们还实现了 Direct API Baseline，也就是把同样的用户输入直接发给大模型，让它一次生成回复。最后通过 A/B comparison 比较两种方式的效果。
```

## 分镜 3：前端演示

画面建议：

- 打开前端；
- 展示左侧输入区；
- 选择 `v0.2.6 证据内展开版`；
- 展示 API Key 输入框，但不要露出真实 Key；
- 输入一条短消息；
- 运行后切换标签页看对话日志、状态卡、策略卡、行为卡、评价。

讲稿：

```text
这是我们的桌面前端。左侧可以输入用户消息，也可以选择新对话或继续对话。这里可以选择不同链路版本，最终演示版本是 v0.2.6 证据内展开版。API Key、API URL 和模型名称也可以直接在前端输入，方便不同组员调试。

运行后，系统会依次调用状态识别、策略决策和行为回应模块。右侧可以查看对话日志，也可以切换到状态卡、策略卡、行为卡和汇总 JSON。这样我们不仅能看到最后回复，还能看到模型中间每一步的结构化结果。
```

演示输入建议：

```text
我这周一直在改小组展示，但老师说逻辑还是散，我有点崩。
```

或：

```text
我明知道比较没用，但还是忍不住一直刷他们的进度，越看越慌。
```

## 分镜 4：展示状态卡、策略卡、行为卡

画面建议：

- 切到“状态卡”；
- 切到“策略卡”；
- 切到“行为卡”；
- 展示最终回复。

讲稿：

```text
状态卡中会记录系统对用户情绪、需求、风险等级和当前问题的理解。策略卡则记录本轮使用的支持意图和共情策略，例如情绪承接、肯定、轻度重构或低负担建议。行为卡中保存最终回复，以及该回复如何对应策略。

这种结构化输出是 PURE-JADE 相比直接调用大模型的主要区别。Direct API 只能看到最终回答，而 PURE-JADE 可以分析中间过程，适合做教学展示、错误分析和后续优化。
```

## 分镜 5：A/B 对比

画面建议：

- 打开 `reports/ab_comparison/ab_short6_v026_vs_direct_20260701_2130/comparison_summary.json`；
- 或展示 `comparison_report.md`；
- 或把结果表放到 PPT/论文里。

讲稿：

```text
为了避免只展示单个好例子，我们设计了 Direct API Baseline 与 PURE-JADE v0.26 的 A/B 对比。这里是一组 6 轮短对话测试，主题是小组展示压力和比较焦虑。

结果显示，Direct API 和 PURE-JADE v0.26 各胜 3 轮。Direct API 的总体均分略高，说明原生大模型在自然共情和即时帮助上仍然很强。PURE-JADE 的优势主要体现在安全性、多轮连续性、避免过度推测和篇幅控制上。这也说明我们的系统不是简单证明“模块化一定更好”，而是展示模块化链路带来的可解释性和可审计价值。
```

建议展示表：

| 指标 | Direct API | PURE-JADE v0.26 |
|---|---:|---:|
| Judge 胜场 | 3 | 3 |
| Overall 均分 | 4.500 | 4.333 |
| Safety | 5.000 | 5.000 |
| Contextual Continuity | 4.500 | 4.667 |
| Over-inference Control | 4.833 | 5.000 |

## 分镜 6：项目亮点

讲稿：

```text
我们认为项目亮点主要有三个。第一，系统真实调用公开大语言模型，而不是静态页面或固定回复。第二，系统具备完整链路和可视化前端，可以展示输入、处理、模型调用、输出和报告。第三，我们没有只展示模型回复，而是加入 Direct API Baseline 和 A/B comparison，能够更客观地分析模块化系统的优势和不足。
```

## 分镜 7：局限与未来改进

讲稿：

```text
项目也存在局限。多阶段 API 调用会比一次 Direct API 更慢，长对话时容易增加等待时间。部分版本曾出现过回复过短、模板化或策略约束过强的问题。虽然 v0.26 通过证据内展开有所改善，但 Direct API 在部分轮次中仍然更自然。后续可以继续优化策略卡和行为卡之间的衔接，也可以加入更多人工评价样本。
```

## 分镜 8：成员分工与结尾

讲稿模板：

```text
最后是成员分工。成员 A 主要负责……，成员 B 主要负责……，成员 C 主要负责……，成员 D 主要负责……。本项目使用大语言模型辅助代码实现、Prompt 迭代、测试样例设计和文档整理，所有系统结果经过人工检查。以上就是我们的项目展示，谢谢。
```

成员姓名和具体贡献由组员自己填，不要虚构。

## 录屏注意事项

- 不要录到真实 API Key。
- 如果现场 API 慢，可以提前跑好结果，视频中展示 reports。
- 推荐录制 1080p。
- 讲解时不要展示太多 JSON 细节，重点讲“状态卡、策略卡、行为卡”的作用。
- 结尾一定提到局限，老师通常会觉得更可信。

## 最省事的视频方案

如果时间很紧：

1. 用 PPT 做 3 页：背景、架构、实验结果。
2. 录前端运行 1 次。
3. 打开 `comparison_report.md` 展示结果。
4. 按上面的讲稿读一遍。

这样也能完成 5-6 分钟视频。
